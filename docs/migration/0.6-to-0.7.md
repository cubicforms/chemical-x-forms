# Migrating from 0.6.x to 0.7.x

Two breaking changes. The second is the one that actually changes
application code.

## `handleSubmit` now returns a handler, not a Promise

**0.6:** `handleSubmit(cb)` ran the submit synchronously and returned
the resulting Promise. Consumers chained on it:

```ts
// OLD — 0.6
const onSubmit = () => handleSubmit(async (values) => {
  await api.post('/signup', values)
})
```

**0.7:** `handleSubmit(cb)` returns the handler function. The consumer
binds it directly to `@submit.prevent` or calls it imperatively:

```ts
// NEW — 0.7
const onSubmit = handleSubmit(async (values) => {
  await api.post('/signup', values)
})
```

```vue
<template>
  <!-- OLD — 0.6 -->
  <form @submit.prevent="onSubmit">...</form>
  <!-- NEW — 0.7: identical template! -->
  <form @submit.prevent="onSubmit">...</form>
</template>
```

The template doesn't change — what changes is the value of `onSubmit`.
Under 0.7 it's a function you bind or call; under 0.6 it was a thunk
that ran `handleSubmit` each time it was invoked. The new shape
composes with `preventDefault` more naturally (the returned handler
accepts an `Event` directly) and avoids the "what happens if I bind
the Promise to a template event handler" footgun.

If you had `onSubmit()` expressions in code, keep them — calling the
returned function is still idiomatic.

## The type surface tightened

No runtime change, but the TypeScript types around
`UseAbstractFormReturnType` were narrowed. If you were casting to
`any` around `register` or `handleSubmit`, the cast is likely no
longer necessary — try removing it and see if the compiler agrees.

The next breaking change (required `key`) came in 0.7.2. See
[`0.7-to-0.8.md`](./0.7-to-0.8.md) if you're jumping straight to 0.8.
