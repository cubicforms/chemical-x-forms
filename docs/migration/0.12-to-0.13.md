# Migrating from 0.12.x to 0.13.x

Three themes:

- **Pinia-style read API.** `getValue`, `getFieldState`, and
  `fieldErrors` are gone — replaced by `form.values`,
  `form.fieldState`, and `form.errors`, which read identically in
  templates and scripts (no `.value`, no auto-unwrap rules).
- **Persist contradictions throw at construct time.** Two
  configurations that previously logged a microtask warning now
  throw `AnonPersistError` from the `useForm()` / `register()`
  call site, with the form's schema fields and a captured call-site
  in the error body.
- **Blank terminology.** `transientEmpty` / `pendingEmpty`
  collapsed to `blank` everywhere — meta flag, `FieldState` field,
  the form-level set, the persistence wire format.

## Breaking changes

1. **`getValue` removed.** Read from `form.values` directly:

   ```diff
   - const email = form.getValue('email')           // Readonly<Ref<string>>
   - const upper = email.value.toUpperCase()
   + const upper = form.values.email?.toUpperCase() ?? ''
   ```

   ```diff
   - <p>{{ form.getValue('email') }}</p>
   + <p>{{ form.values.email }}</p>
   ```

   Read paths that cross an array index taint with `| undefined` —
   narrow with optional chaining. For ref-shaped interop (`watch()`
   / external composables), use `form.toRef('email')` to get a
   `Readonly<Ref<...>>` at the same path.

2. **`getFieldState` removed.** Read from `form.fieldState`:

   ```diff
   - const state = form.getFieldState('email')
   - if (state.value.dirty) save()
   + if (form.fieldState.email.dirty) save()
   ```

   ```diff
   - <small v-if="form.getFieldState('email').value.errors.length">
   -   {{ form.getFieldState('email').value.errors[0].message }}
   - </small>
   + <small v-if="form.fieldState.email.errors.length">
   +   {{ form.fieldState.email.errors[0].message }}
   + </small>
   ```

   Leaf keys: `value`, `original`, `dirty`, `pristine`, `focused`,
   `blurred`, `touched`, `errors`, `blank`, `isConnected`,
   `updatedAt`, `path`. Schema field names matching one of these at
   depth ≥ 2 are shadowed; rename the schema field or use
   `form.toRef(path)` if the collision matters.

3. **`fieldErrors` renamed to `errors`.** Same Proxy shape; same
   dot-access semantics. Schema entries first, user entries second.

   ```diff
   - <small v-if="form.fieldErrors.email?.[0]">{{ form.fieldErrors.email[0].message }}</small>
   + <small v-if="form.errors.email?.[0]">{{ form.errors.email[0].message }}</small>
   ```

   ```diff
   - watch(() => form.fieldErrors.email, (next) => …)
   + watch(() => form.errors.email, (next) => …)
   ```

4. **`transientEmpty` / `pendingEmpty` → `blank`.** The
   per-field meta flag, the form-level set, the `FieldState` field,
   and the persistence wire envelope all rename:

   ```diff
   - if (form.getFieldState('income').value.pendingEmpty) …
   + if (form.fieldState.income.blank) …
   ```

   ```diff
   - for (const key of form.transientEmptyPaths.value) …
   + for (const key of form.blankPaths.value) …
   ```

   Persisted drafts written under 0.12 (envelope `v: 3` with a
   `transientEmptyPaths` field) auto-invalidate via the schema's
   structural fingerprint — no manual migration needed.

5. **`useForm({ persist })` without `key:` throws.**
   Anonymous synthetic keys (`__cx:anon:*`) drift across mounts;
   persisting to a non-deterministic location is now refused at
   `useForm()` construction.

   ```diff
   - useForm({ schema, persist: 'session' })
   + useForm({ schema, persist: 'session', key: 'signup' })
   ```

   The throw is `AnonPersistError` with `cause: 'no-key'`, the
   schema's top-level fields, and a captured call-site in the
   message body. Throws in dev and prod alike.

6. **`register({ persist: true })` on a form without `persist:`
   throws.** Previously a one-shot `console.warn`; now an eager
   `AnonPersistError` with `cause: 'register-without-config'`.
   Either set `persist:` on `useForm()` or drop `{ persist: true }`
   from the offending `register()` call:

   ```diff
   - const form = useForm({ schema, key: 'signup' })
   - <input v-register="form.register('email', { persist: true })" />
   + const form = useForm({ schema, key: 'signup', persist: 'session' })
   + <input v-register="form.register('email', { persist: true })" />
   ```

   Persistence is operational only at the form level — no field
   ever auto-persists from the form-level config alone — so the two
   layers must align.

7. **`useRegister` SSR diagnostic moved to `onMounted`.** The
   no-parent-`v-register` warn fires once per instance at
   `onMounted`, not on every read of the returned computed.
   Intentionally silent during `renderToString` — Vue skips
   `onMounted` on the server, and the CSR hydration pass surfaces
   the same diagnostic without double-counting through Nuxt's
   `dev:ssr-logs` channel.

   No code change needed unless you were asserting on the warn count
   in tests — expect exactly one warn per instance now (not one per
   `register.value` read).

## What didn't change

- `register()`, `setValue()`, `handleSubmit()`, `validate()`,
  `validateAsync()`, `reset()`, `resetField()`, `undo()` / `redo()`,
  the field-array helpers, `setFieldErrors` / `addFieldErrors` /
  `clearFieldErrors`, `parseApiErrors`, the `unset` sentinel, the
  `v-register` directive
- The slim-write contract (writes accept the slim primitive type;
  refinements surface as field errors)
- The structural-completeness invariant on `setValue`
- SSR via `renderChemicalXState` / `hydrateChemicalXState`
- Nuxt module wiring

## Sweep checklist

- [ ] Replace every `form.getValue(...)` script-side read with
      `form.values.<path>` (or `form.toRef(path)` for ref-shape).
- [ ] Replace every `form.getFieldState(...)` read with
      `form.fieldState.<path>.<leaf>`.
- [ ] Rename `form.fieldErrors` → `form.errors`.
- [ ] Rename `transientEmpty` / `pendingEmpty` references to
      `blank`; rename `transientEmptyPaths` → `blankPaths`.
- [ ] Audit `useForm({ persist })` calls for missing `key:`. Audit
      `register({ persist: true })` calls for missing form-level
      `persist:` config. Wire one or the other; remove the
      contradiction.
- [ ] Drop any test assertion on a multi-fire `useRegister` warn —
      expect exactly one warn per instance.
