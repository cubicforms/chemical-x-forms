# Migrating from 0.13.x to 0.14.x

The `useForm` return surface is reshaped around a single coherent
shape: three drillable, callable, leaf-aware proxies (`values`,
`errors`, `fields`) plus one form-wide aggregate (`meta`). Two
themes:

- **Leaf-aware proxy resolution.** `form.fields.<path>` resolves to a
  `FieldStateLeaf` only at LEAF paths (primitives, dates). At
  container paths the proxy descends without injecting reserved keys
  — a schema field literally named `dirty` at depth 2+ is now
  reachable as a descent target. The old shadowing edge case is gone.
- **`state` → `meta`.** `form.state` becomes `form.meta`. The new
  bundle adds `meta.errors` — a flat `ValidationError[]` of every
  error in the form, useful for "show all" UIs.

## Breaking changes

1. **`form.state` → `form.meta`.** Same fields, plus a new
   `errors` aggregate.

   ```diff
   - <button :disabled="form.state.isSubmitting">Save</button>
   + <button :disabled="form.meta.isSubmitting">Save</button>
   ```

   ```diff
   - watch(() => form.state.isValid, (next) => …)
   + watch(() => form.meta.isValid, (next) => …)
   ```

   New on `form.meta`:

   ```ts
   form.meta.errors // readonly ValidationError[] — every error in the form
   ```

   Use `form.meta.errors` for "show all" UIs:

   ```vue
   <ul v-if="form.meta.errors.length">
     <li v-for="err in form.meta.errors" :key="err.path.join('.')">
       {{ err.path.join('.') || 'form' }}: {{ err.message }}
     </li>
   </ul>
   ```

   `meta.errors` is UNFILTERED — inactive-variant errors stay in the
   array. Use `form.errors.<path>` (per-leaf, active-path filtered)
   when you want the variant-aware view.

2. **`useFormContext` → `injectForm`.** The verb tells the reader
   which side of the provide/inject pair they're on (`useForm`
   creates and provides; `injectForm` looks up).

   ```diff
   - import { useFormContext } from '@chemical-x/forms'
   - const form = useFormContext<SignupShape>('signup')
   + import { injectForm } from '@chemical-x/forms'
   + const form = injectForm<SignupShape>('signup')
   ```

   `null`-on-miss return + dev-warn unchanged. Same arg semantics
   (no key → ambient ancestor; key → addressable form).

3. **`form.errors` shape: flat record → drillable callable proxy.**
   The old flat-record (`Partial<Record<FlatPath<Form>,
ValidationError[]>>`) is gone. Drill via dot/bracket OR call
   dynamically:

   ```diff
   - form.errors['address.city']           // single-bracket dotted (old flat record)
   + form.errors.address.city              // chained drill
   + form.errors('address.city')           // callable form (dynamic / programmatic)
   + form.errors(['address', 'city'])      // path-array form
   ```

   Single-bracket dotted access (`form.errors['user.profile.email']`)
   is intentionally NOT supported — JS object semantics treat the
   dotted string as a single key. Use chained dot/bracket or the
   callable form.

   `Object.keys(form.errors)` no longer enumerates error keys —
   containers are descend-only. Use `form.meta.errors` for the
   "every error" aggregate.

4. **`form.fields` shape: leaf-aware drillable callable proxy.**
   `FIELD_STATE_KEYS` (`value`, `dirty`, `errors`, `isConnected`,
   …) inject ONLY at leaf paths now. Schema field collisions at
   depth 2+ resolve unambiguously:

   ```diff
   // schema: { address: { isValid: z.boolean(), city: z.string() } }
   - form.fields.address.isValid          // (0.13: undefined — schema field shadowed by leaf-key)
   + form.fields.address.isValid          // FieldStateView for the leaf (address.isValid)
   + form.fields.address.isValid.value    // boolean (the schema field's current value)
   + form.fields.address.isValid.isValid  // (no such leaf-key — typo or future addition)
   ```

   Container reads of leaf-key names (`form.fields.address.dirty`
   where `dirty` ISN'T a schema field) now descend instead of
   shadowing. The "any-descendant dirty" boolean is gone — compute
   it yourself if you need it:

   ```ts
   // 0.13: form.fields.address.dirty was "any descendant dirty"
   // 0.14: compute the aggregate
   const anyDirty = form.meta.errors.some((e) => e.path[0] === 'address')
   // (or walk form.fields.address.* via known leaf paths)
   ```

   Callable form available too:

   ```ts
   form.fields.email.errors // dot/bracket
   form.fields('email').errors // callable form
   form.fields(['users', 0, 'name']) // path-array
   form.fields() // root proxy
   ```

5. **`form.values` is callable.** Same dot-access as before, plus
   the call shape:

   ```ts
   form.values.address.city // unchanged
   form.values('address.city') // NEW: function-call (dynamic / programmatic)
   form.values(['address', 'city']) // NEW: path-array
   form.values() // NEW: returns the whole form (root)
   ```

   `JSON.stringify(form.values)` works (a `toJSON` short-circuit
   serialises the readonly inner proxy — the typeof === 'function'
   surface is invisible here).

6. **Migration shortcut for `Object.keys(form.errors).length`.**
   Use `form.meta.errors.length` instead.

   ```diff
   - <span>{{ Object.keys(form.errors).length }} issue(s)</span>
   + <span>{{ form.meta.errors.length }} issue(s)</span>
   ```

7. **`FormState` type → `FormMeta`.** Type imports rename:

   ```diff
   - import type { FormState } from '@chemical-x/forms'
   + import type { FormMeta } from '@chemical-x/forms'
   ```

8. **`FormFieldErrors<Form>` shape changes.** No longer
   `Partial<Record<FlatPath<Form>, ValidationError[]>>`. Now a
   leaf-aware callable proxy type. Most consumers just read
   per-leaf and never reference the type explicitly — if you DO
   import it, the new shape is `FormErrorsSurface<Form>`.

9. **`getAtPath(form.values, path)`** patterns.
   `form.values` is `typeof === 'function'` post-rewrite; helpers
   that descend via property access on the proxy keep working
   (the `get` trap handles them), but generic helpers that probe
   `typeof === 'object'` (like the library's internal `getAtPath`)
   need an unwrap:

   ```diff
   - getAtPath(form.values, ['users', 0, 'email'])
   + getAtPath(form.values(), ['users', 0, 'email'])
   ```

   Or use the callable form directly:

   ```ts
   form.values(['users', 0, 'email']) // unknown — same value, no helper needed
   ```

## What didn't change

- `form.register`, `form.setValue`, `form.handleSubmit`,
  `form.validate` / `validateAsync`, `form.reset` / `resetField`,
  `form.toRef`, `form.setFieldErrors` / `addFieldErrors` /
  `clearFieldErrors`, `form.persist` / `clearPersistedDraft`,
  `form.focusFirstError` / `scrollToFirstError`, `form.undo` /
  `redo`, the field-array helpers (`append`, `prepend`, `insert`,
  `remove`, `swap`, `move`, `replace`), `form.blankPaths`,
  `form.key`.
- The slim-write contract (writes accept the slim primitive type;
  refinements surface as field errors).
- The structural-completeness invariant on `setValue`.
- SSR via `renderChemicalXState` / `hydrateChemicalXState`.
- Nuxt module wiring.
- The active-path filter on `form.errors.<path>` (preserved
  semantics — inactive-variant errors hidden from the per-leaf
  view; `form.meta.errors` shows them all).

## Sweep checklist

- [ ] Replace every `form.state` reference with `form.meta`.
- [ ] Replace every `useFormContext` import / call with
      `injectForm`.
- [ ] Convert `form.errors['some.dotted.path']` to chained access
      (`form.errors.some.dotted.path`) or the callable form
      (`form.errors('some.dotted.path')`).
- [ ] Replace `Object.keys(form.errors).length` with
      `form.meta.errors.length`.
- [ ] Audit schema fields named `value`, `dirty`, `errors`,
      `touched`, `isConnected`, `blank`, `path`, `pristine`,
      `focused`, `blurred`, `original`, `updatedAt` at depth 2+
      — they're now reachable via descent (instead of shadowed).
      No code change required if you weren't relying on the
      shadow.
- [ ] If you import `FormState` / `FormFieldErrors` types, update
      to `FormMeta` / `FormErrorsSurface`.
- [ ] If you wrote helpers that pass `form.values` to functions
      checking `typeof === 'object'` (e.g. `getAtPath` from the
      library, or your own deep-equal), unwrap with
      `form.values()` or use the callable form.
